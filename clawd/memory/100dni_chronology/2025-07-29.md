  Aby workflow s GitHub Trigger node fungoval, potrebuješ byť v GitHub organizácii aspoň adminom daného repozitára alebo ownerom organizácie.
  
  Detailné požiadavky na práva: 
  
   1. Admin práva na repozitári
  
   ▫ Musíš byť „Admin“ v konkrétnom repozitári, pre ktorý nastavuješ webhook (cez GitHub Trigger node).
  
   ▫ Len admin môže vytvárať, upravovať alebo mazať webhooky, ktoré sú potrebné na to, aby GitHub Trigger node fungoval.
  
   2. Owner v organizácii (voliteľné, ale výhodné)
  
   ▫ Ak si owner celej organizácie, máš automaticky admin práva na všetky repozitáre a vieš nastavovať webhooky kdekoľvek.
  
   3. Personal Access Token (PAT) alebo OAuth credentials
  
   ▫ Token, ktorý používaš v n8n, musí byť vytvorený používateľom s admin právami na repozitári/organizácii.
  
   ▫ Token musí mať scope ‎`repo` (pre private repozitáre) a ideálne aj ‎`admin:repo_hook`.
  
  Čo nestačí: 
  
   • Bežný člen organizácie bez admin práv na repozitári nemôže nastavovať webhooky.
  
   • Token s obmedzenými scopes (napr. len ‎`read`) nebude fungovať pre webhooky.
  
  Zhrnutie pre workflow: 
  
   • Musíš byť admin repozitára alebo owner organizácie.
  
   • Token v n8n musí byť viazaný na účet s týmito právami a mať správne scopes.
  
   • Len tak bude GitHub Trigger node schopný vytvoriť a spravovať webhooky a workflow bude fungovať.
  
  Ak potrebuješ presný postup, ako si overiť svoje práva alebo nastaviť token, daj vedieť!
  
  Presne tak – ak nemáš admin práva na repozitári, nemôžeš webhook otestovať, lebo GitHub Trigger node nebude prijímať eventy.
  
  Prečo to tak je: 
  
   • n8n GitHub Trigger node vytvára webhook v repozitári, aby mohol počúvať na eventy (napr. Pull Request Review).
  
   • Webhook sa vytvorí len vtedy, ak máš admin práva na danom repozitári (alebo si owner organizácie).
  
   • Ak tieto práva nemáš, webhook sa nevytvorí a n8n node bude len „listening“ bez toho, aby niekedy niečo prijal – aj keď na GitHube spravíš akciu, workflow sa nespustí.
  
  Čo s tým? 
  
   • Nemôžeš workflow otestovať na reálnych GitHub eventoch, kým nemáš admin/owner práva.
  
   • Môžeš workflow testovať len s mock/test dátami (napr. cez Set/Function node), ale nie s reálnym GitHub Triggerom.
  
  Čo povedať Peťovi: 
  
  Peťo, nemám admin práva na repozitári, takže nemôžem otestovať webhook ani GitHub Trigger node v n8n.
  
  Potrebujem, aby si mi dal admin práva na repozitári (alebo owner v organizácii), alebo aby si webhook nastavil ty cez svoj účet.
  
  Inak workflow neviem otestovať ani nasadiť na reálne eventy z GitHubu.
  
  Zhrnutie:
  
  Bez admin práv na repozitári nemáš ako overiť, že GitHub Trigger node v n8n funguje – workflow bude stále len „listening“ a nikdy sa nespustí.
